2
Project Setup & Infrastructure
Preparar o terreno — feito uma vez, serve o projecto inteiro · Semana 1 de dev
PENDENTE
▶
Estrutura de pastas — Clean Architecture
Cria: src/domain/, src/application/use-cases/, src/infrastructure/database/, src/interfaces/http/. Esta estrutura garante que a lógica de negócio nunca depende de Express ou pg directamente.
TypeScript + ESLint + Prettier + Husky
Configura tsconfig.json com strict: true. ESLint para apanhar erros. Prettier para formatação automática. Husky para bloquear commits com erros. Faz isto antes de escrever código de negócio.
Git Strategy + Conventional Commits
Branches: main (produção), develop (integração), feature/nome-feature. Commits: feat(stock): add low stock alert. Isto cria um histórico limpo que impressiona em portfólio.
Docker Compose — PostgreSQL local
docker-compose.yml com serviço postgres:16. Nunca instales PostgreSQL directamente na máquina — Docker garante que o ambiente é reproduzível e não polui o sistema.
Migrations SQL manuais
Como usas raw SQL com pg, crias os ficheiros de migration manualmente: 001_create_users.sql, 002_create_products.sql. Cada migration é irreversível para baixo — pensa bem antes de correr.
3
Core Backend Development
Domain → Application → Infrastructure → Interfaces · Semanas 2–14 de dev
PENDENTE
▶
Auth — JWT + RBAC
POST /auth/login → verifica password com bcrypt → gera accessToken (15min) + refreshToken (7 dias). Middleware que valida token em cada route protegida. Roles: admin, manager, warehouse, hr, accountant, sales.
Domain Layer — Entidades puras
Classes TypeScript com validação interna: Product.create() lança erro se nome < 2 chars. product.decreaseStock() lança erro se stock insuficiente. Zero imports de Express, pg ou qualquer biblioteca externa.
Repository Layer — raw SQL com pg
ProductRepository implementa IProductRepository usando Pool do pg. Queries parametrizadas sempre: query('SELECT * FROM products WHERE id = $1', [id]). Nunca concatenas strings em SQL.
Use Cases por módulo
Cada Use Case faz uma única coisa: CreateProductUseCase, UpdateStockUseCase, ProcessSalaryUseCase, CreateSaleUseCase. Recebe DTO, orquestra Domain + Repository, retorna resultado.
Controllers + Validação com Zod
Controller não tem lógica de negócio. Recebe request → valida com Zod → chama Use Case → retorna response. Se Zod falha, responde 400 automaticamente.
Error Handling centralizado
Middleware global de erros no final da app Express. AppError (400), NotFoundError (404), ConflictError (409). Erros inesperados retornam 500 sem expor detalhes ao cliente.
4
Frontend Development
React + TypeScript + React Query · Semanas 15–20 de dev
PENDENTE
▶
Setup React + TypeScript + React Router + Tailwind
Estrutura: src/features/stock/, src/features/hr/, src/components/ui/, src/lib/api/. Um router protegido que verifica se há token antes de renderizar qualquer página.
Axios + interceptors + React Query
Interceptor de request: adiciona Bearer token. Interceptor de response: se 401, tenta refresh. Se falha o refresh, logout. React Query: useQuery para GET, useMutation para POST/PUT/DELETE.
UI por módulo — Stock → Vendas → RH → Finanças
Um módulo de cada vez. Tabelas, formulários, dashboards com KPIs. Segue Atomic Design: Atoms (Button, Input) → Molecules (SearchInput) → Organisms (ProductTable) → Pages.
5–6
Testing, Deploy & Observability
Qualidade e produção · Semanas 21–24 de dev
PENDENTE
▶
Testes unitários — Vitest
Testa Use Cases com repositórios mock. Exemplo: CreateProductUseCase com MockProductRepository. Não precisas de base de dados para testar lógica de negócio — essa é a vantagem da Clean Architecture.
Segurança OWASP — Helmet + Rate Limiting + CORS
helmet() nos headers. express-rate-limit nos endpoints públicos. CORS configurado só para o domínio do frontend. UUID em vez de IDs sequenciais. Nunca retornas passwords mesmo em hash.
Docker + Deploy + Winston Logging
Dockerfile multi-stage. docker-compose para produção. Winston com logs em JSON (fácil de pesquisar). GET /health que verifica DB. Deploy inicial no Railway (~$5/mês) para demo ao cliente.
